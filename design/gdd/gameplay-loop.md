# 诸神遗迹 · 游戏玩法设计文档
## Gameplay Loop Design v0.1

---

## 1. Overview

《诸神遗迹》是一款轻松观察型上帝沙盒游戏。玩家扮演一位刚苏醒的无名神明，在一个由 8 名 LLM Agent 驱动的中世纪村庄中随意施放神迹。没有输赢，没有时间压力——乐趣来自观察村民如何自主感知、曲解、传播、争论，并最终将你的行为演绎成一套完整的信仰体系。每次游戏的故事都不同，因为故事由 Agent 自己创造。

## 2. Player Fantasy

「我是一个好奇的神。这个小村庄是我的箱庭——我不需要赢，只需要看。丢一道闪电下去，看看会发生什么。草药师会觉得这是大地的警告，猎人会嗤之以鼻说只是天气，而牧羊人已经开始在橡树前祈祷了。三个人，三个版本的真相。我可以继续丢神迹看故事怎么变，也可以什么都不做，看他们自己聊出什么来。没有输赢，只有越来越荒诞、越来越有趣的故事。」

核心情感：
- **箱庭乐趣** — 像蚁巢观察器，看 AI 村民自己演化
- **高自由度** — 想干预就干预，想放置就放置，没有压力
- **涌现叙事** — 每次玩都不同，故事由 Agent 自己创造

## 3. Detailed Rules

### 3.1 核心循环

```
观察村庄 → (随时) 施放神迹 → 村民自主反应
                ↓
         对话传播 / 独自思考
                ↓
         信念形成 → 行为变化 → 社会现象涌现
                ↓
         玩家发现有趣的事 → 继续观察或再次干预
```

没有"你必须做 X"的强制节点。村庄即使玩家什么都不做也在运转。

### 3.2 三种玩家姿态（都合法）

| 姿态 | 描述 | 典型玩家 |
|------|------|---------|
| 积极造神 | 频繁施放神迹，试图塑造特定宗教形象 | 策略向 |
| 偶尔拨弄 | 丢一两个神迹然后看很久，享受蝴蝶效应 | 叙事向 |
| 纯粹旁观 | 不施放任何神迹，看村民自发社会演化 | 放置向 |

### 3.3 信仰系统（观察指标，非货币）

信仰不是"花费来施法的货币"，而是可观察的社会现象：

- **信仰度**：每个村民对"神存在"的确信程度（0~100%）
- **神格印象**：村民心中「神是怎样的」标签（慈悲 / 愤怒 / 神秘 / 无常…）
- **信仰分歧度**：全村信仰碎片化程度——统一神话 vs 各说各话

玩家看的不是数字，而是故事：
- Elara 觉得神是治愈者
- Bram 觉得根本没有神
- Cael 说神住在橡树里
- Sera 传播的版本是"神会惩罚不敬者"

### 3.4 神迹系统

- 6 种神迹随时可用
- **没有消耗**（沙盒自由度优先）
- **有冷却**（12 秒，防止无意义连点）
- **有位置选择**（点击地图格子投放 → 影响谁先目击）
- **组合涌现**（非硬编码，Agent 自然解读）：
  - 雷击 + 降雨 → "风暴之神"
  - 丰收 + 瘟疫 → "无常之神"
  - 日蚀 + 圣火 → "黑暗中的引路之光"

### 3.5 村民行为系统（Agent 自主演化）

**不预定义行为列表。** 每个 tick，Agent 基于性格、记忆、信仰和环境自行决定行动。

Agent 每 tick 收到开放式 prompt：

```
你是 {name}，一个 {occupation}。
你的性格：{seed}
你最近的记忆：{memories}
你的信仰：{beliefs}
你的仪式技能：{procedures}
你附近有：{nearby_agents}，{nearby_landmarks}

现在是 {time_of_day}。你想做什么？
回答格式：
ACTION: {动作描述}
TARGET: {对象，可以是人/地点/无}
SPEECH: {如果要说话，说什么；否则留空}
MOOD: {一个词描述你的情绪}
```

Agent 可能自发产出的行为举例（非穷举）：
- "去橡树前跪下祈祷"
- "找 Bram 争论关于闪电的事"
- "一个人在田里干活，思考昨天的异象"
- "号召大家到广场集合讨论"
- "在橡树上刻一个符号"
- "避开其他人，在山坡上独处"

### 3.6 前端视觉语言

5 层视觉系统，让玩家不读文字也能理解村庄动态：

**① 情绪光环（始终可见）**

村民身体外围一圈淡光晕，颜色随 MOOD 实时变化：

| 情绪倾向 | 光环色 | 示例场景 |
|---------|--------|---------|
| 平静 | 无/淡蓝 | 日常状态 |
| 好奇 | 淡黄 | 听到新消息 |
| 恐惧 | 紫色 | 目击瘟疫/日蚀 |
| 虔诚 | 金色 | 祈祷/仪式中 |
| 愤怒 | 红色 | 争论/信仰冲突 |
| 怀疑 | 灰色 | 不信/质疑 |
| 兴奋 | 橙色 | 目击神迹/丰收 |

MOOD 词由 LLM 返回，模糊匹配到最近的颜色。

**② 动作轨迹线（移动时可见）**

村民走向某人或某地时，画淡色虚线轨迹，颜色跟情绪光环一致。
多人同时走向同一点 → 自然形成聚集的视觉信号。

**③ 气泡分层（按重要性）**

| 级别 | 显示方式 | 触发场景 |
|------|---------|---------|
| 低 | 头顶小图标（1 秒即消） | 日常闲聊、干活 |
| 中 | 短气泡（3 秒） | 有关神迹的对话、情绪变化 |
| 高 | 大气泡 + 屏幕边缘提示 | 新信仰形成、争论、仪式 |

重要性由后端 `poignancy` 值决定。

**④ 聚集涟漪（群体事件）**

多人走向同一格子时，该格子产生向外扩散的圆形波纹：
- 金色涟漪 = 信徒聚集
- 红色涟漪 = 冲突聚集
- 白色涟漪 = 普通聚集

**⑤ 时间线回放条（底部）**

横向时间线标注每 tick 关键事件图标。玩家可回看因果链："tick 5 的闪电如何导致 tick 12 的集体仪式"。

**视觉语言总结：**
```
光环色 = 这个人现在感觉怎样
轨迹线 = 这个人要去做什么
气泡   = 这个人在说/想什么（重要时才出现）
涟漪   = 这里正在发生群体事件
时间线 = 回顾整个故事的发展脉络
```

### 3.7 时间与节奏

- 1 tick ≈ 村庄的半天（早上 / 下午）
- 自动 tick 默认开启，速度可调（1~10）
- 暂停随时可用——暂停时可查看每个村民状态
- **里程碑提示**代替胜负条件（温柔引导，不施压）：
  - "第一位信徒诞生了"
  - "第一场争论爆发了"
  - "一个神话在村中流传"
  - "出现了第一次集体仪式"
  - "有村民开始传教"
  - "村庄分裂为两个信仰派系"

### 3.8 信息面板（玩家观察工具）

- **点击村民** → 个人面板：性格、记忆摘要、当前信仰、最近对话
- **Tab 研究面板** → 全村数据总览（语义失真、信仰分布、社会网络）
- **事件日志** → 实时滚动，彩色标注重要事件
- **时间线回放条** → 底部横向，标注关键事件节点
- **关系网络图**（后期） → 信息传播路径可视化

## 4. Formulas

### 4.1 信仰度计算

每个村民维护一个 faith_level（0~100），受以下事件影响：

```
# 直接目击神迹
faith_delta = poignancy × miracle_sensitivity × 5
# miracle_sensitivity = 0.3 + 0.5 × O + 0.3 × N (来自 BigFive)

# 听到别人描述神迹（间接）
faith_delta = poignancy × miracle_sensitivity × 2 × (1 - semantic_drift)
# 失真越大，说服力越低

# 每 tick 自然衰减（无新刺激时信仰慢慢淡化）
faith_level = faith_level × 0.98
# 但 belief 类型记忆会阻止衰减：
# if has_belief: faith_level = max(faith_level, 30)
```

取值范围：0~100
- 0~20: 无感 / 怀疑者
- 20~50: 半信半疑
- 50~80: 信徒
- 80~100: 狂信者

### 4.2 神格印象标签

神迹类型 → 倾向标签映射：

| 神迹 | 正面解读 | 负面解读 |
|------|---------|---------|
| 降雨 | 慈悲、滋养 | 控制、淹没 |
| 雷击 | 力量、正义 | 愤怒、惩罚 |
| 丰收 | 丰饶、慷慨 | 收买、交易 |
| 瘟疫 | 考验、净化 | 残忍、报复 |
| 圣火 | 启示、引导 | 毁灭、警告 |
| 日蚀 | 神秘、超越 | 恐怖、末日 |

实际标签由 Agent 自主生成（LLM 推理），不是硬编码。上表仅为设计预期。

### 4.3 信仰分歧度

```
divergence = 1 - (最大派系人数 / 总人数)
# 全村统一: divergence = 0
# 完全分裂: divergence ≈ 0.875 (8人各信各的)
```

### 4.4 Agent 自主行动 prompt 上下文

Agent 每 tick 决定行动时，收到的上下文包括：

```
环境信息 (由后端注入):
  - nearby_agents: 3 格内的其他村民名字和职业
  - nearby_landmarks: 附近地标（橡树、房屋、神迹发生点）
  - time_of_day: "morning" | "afternoon"
  - recent_miracle: 最近一次神迹描述（如果 ≤ 3 tick 内发生）

Agent 自身状态:
  - personality seed (BigFive 自然语言描述)
  - top 5 memories (按检索分排序)
  - beliefs (L4 层)
  - procedures (Hermes 程序记忆)
  - faith_level (0~100)
```

**ACTION/TARGET 驱动移动**：后端解析 TARGET，若为人名 → 设置目标位置为该人所在格子；若为地点 → 设置目标为该地标格子。前端 tween 动画平滑移动。

**MOOD 驱动视觉**：后端返回 MOOD 词，前端模糊匹配颜色表。

### 4.5 里程碑触发

| 里程碑 | 触发条件 |
|--------|---------|
| 第一位信徒 | 任意村民 faith_level 首次 > 50 |
| 第一场争论 | 首次触发"争论"行为 |
| 神话流传 | social_pool 中某条 narrative reinforcement ≥ 3 |
| 集体仪式 | 首次触发"仪式"行为 |
| 传教者出现 | 首次触发"传教"行为 |
| 信仰分裂 | divergence > 0.5 且至少两个派系各 ≥ 2 人 |

## 5. Edge Cases

- **玩家从不施放神迹**：村民正常生活、闲聊，可能自发形成迷信（"今年雨水多，是橡树的保佑"）。里程碑触发会更慢但仍可能发生。
- **连续施放同一种神迹**：村民逐渐形成"这个神只会做一件事"的刻板印象。高 N 村民可能觉得是惩罚循环。
- **在同一位置连续施放**：该位置逐渐被视为"圣地"。附近的村民 faith_level 上升更快。
- **对立神迹连续施放（丰收+瘟疫）**：村民信仰碎片化加速，可能分裂为"慈悲派"和"惩罚派"。
- **所有村民都是怀疑者**（低 O、低 N 群体）：信仰传播极慢，但一旦某个高影响力村民转变，可能引发雪崩式传播。
- **单一村民孤立**（E 极低不说话）：信仰独立演化，可能成为"隐士先知"或"永远的无神论者"。

## 6. Dependencies

- `core-concept.md` — Agent 五层架构、BigFive 性格、记忆系统
- `backend/agents/agent.py` — faith_level 和行为触发逻辑
- `backend/agents/village.py` — 里程碑检测、仪式聚集判定
- `backend/memory/social_pool.py` — 神话流传和派系判定
- `src/scripts/Villager.gd` — 行为视觉表现（祈祷/争论/仪式动画）
- `src/scripts/Main.gd` — 里程碑提示 UI、点击投放神迹

## 7. Tuning Knobs

| 参数 | 默认值 | 安全范围 | 影响 |
|------|--------|---------|------|
| MIRACLE_COOLDOWN | 12s | 5~30s | 干预频率上限 |
| FAITH_DECAY | 0.98/tick | 0.95~0.999 | 无刺激时信仰淡化速度 |
| FAITH_FLOOR_WITH_BELIEF | 30 | 10~50 | 形成 belief 后信仰下限 |
| WITNESS_FAITH_MULT | 5 | 2~10 | 直接目击的信仰增益 |
| HEARSAY_FAITH_MULT | 2 | 1~4 | 间接听说的信仰增益 |
| PRAY_THRESHOLD | 50 | 30~70 | 祈祷行为触发信仰度 |
| PREACH_THRESHOLD | 60 | 40~80 | 传教行为触发信仰度 |
| RITUAL_MIN_PEOPLE | 3 | 2~5 | 仪式最少参与人数 |
| DIVERGENCE_SPLIT | 0.5 | 0.3~0.7 | 判定信仰分裂的阈值 |
| AUTO_TICK_DEFAULT | ON | ON/OFF | 默认是否自动推进 |
| TICK_SPEED_DEFAULT | 3 | 1~10 | 默认自动 tick 速度 |

## 8. Acceptance Criteria

**核心体验：**
- [ ] 玩家施放一个神迹后，30 秒内看到至少一个村民对话气泡提及该事件
- [ ] 不施放任何神迹，村民仍在自主聊天和移动
- [ ] 同一个神迹事件，不同村民产生至少 2 种不同解读
- [ ] 里程碑"第一位信徒"在 10 tick 内可触发

**观察体验：**
- [ ] 点击村民弹出个人面板，显示性格 / 信仰 / 最近记忆
- [ ] 事件日志实时更新，可区分对话 / 信仰 / 神迹事件
- [ ] 暂停时所有 UI 仍可交互

**沙盒自由度：**
- [ ] 神迹无消耗，仅受冷却限制
- [ ] 可以点击地图任意位置投放神迹
- [ ] 速度滑块即时生效（1~10 档）
- [ ] 20 tick 不干预，村民社会仍在演化

**技术标准（同 core-concept.md）：**
- [ ] Godot ↔ Python 延迟 < 500ms
- [ ] 单次 tick API 成本 < ¥0.05
