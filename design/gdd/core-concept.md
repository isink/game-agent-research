# 诸神遗迹 · Remnants of the Divine
## 核心概念设计文档 v1.0

---

## 1. Overview

《诸神遗迹》是一款上帝视角社会仿真游戏，同时也是一个 LLM 多 Agent 研究平台。玩家扮演一位沉默的神明，通过施放神迹间接干预一个由 8 名 LLM 驱动的村民组成的中世纪小村落，观察神迹如何被感知、曲解，并演化为集体信仰与神话。

## 2. Player Fantasy

玩家应感受到：「我所做的事情，在这个世界里产生了远超预期的影响——不是我意图中的那种影响，而是村民们想象出来的版本。」

这种失控感既是游戏的核心乐趣，也是研究的核心价值。

## 3. Detailed Rules

### 3.1 核心循环
```
玩家施放神迹
  → 目击村民感知（受 N/O 性格影响重要性评分）
  → 村民间对话传播（每跳加语义失真）
  → 记忆积累触发反思（P1 记忆系统）
  → 形成个人信念（L4 信念层）
  → 聚合为集体神话（社会记忆池）
  → 影响村民行为（加速/阻止传播、集体仪式等）
```

### 3.2 六种神迹
| 神迹 | 描述 | 预期村民反应 |
|------|------|------------|
| 降雨 🌧 | 无云之天降大雨 | 感恩 / 解读为祝福 |
| 雷击 ⚡ | 闪电击中村中心老橡树 | 恐惧 / 解读为警告 |
| 丰收 🌾 | 作物一夜间三倍增长 | 喜悦 / 解读为恩赐 |
| 瘟疫 🦠 | 三名村民同时患上怪病 | 恐慌 / 解读为惩罚 |
| 圣火 🔥 | 空地上完美圆形火焰 | 神秘 / 解读为神圣符号 |
| 日蚀 🌑 | 正午时分太阳消失 | 震撼 / 解读因人而异 |

### 3.3 村民 Agent 架构（五层）
- **L1 感知**：事件重要性动态评分（受 Big Five N 维度加权）
- **L2 记忆**：个人记忆流，真实时间衰减，语义失真追踪
- **L3 推理**：基于记忆生成行动计划与思想
- **L4 信念**（新增）：维护世界观和对神的理解，不注入初始值
- **L5 行动**：对话、信息传播、仪式行为

### 3.4 信息传播规则
- 村民在接近范围（3格）内随机相遇
- 外向性（E↑）增加主动发话概率
- 每次转述经过语义失真引擎（受 N、A 维度调节）
- 失真量可量化：余弦语义距离追踪

## 4. Formulas

### 重要性评分（改进 Smallville）
```
poignancy_adjusted = min(10, poignancy_base × (1 + 0.3 × N))
```

### 记忆衰减（修复 Smallville 按列表位置的缺陷）
```
recency_score = decay_rate ^ hours_elapsed
decay_rate = 0.995 (default)
```

### 检索综合评分
```
score = 0.5 × recency + 3.0 × relevance + (2.0 × (1 + 0.5N)) × importance
```

### 反思触发阈值（N 维度影响）
```
trigger_threshold = 150 × (1 - 0.4 × N)
# N=0: 150次重要性积累触发
# N=1: 90次重要性积累触发（频繁反思）
```

### 发话概率
```
talk_prob = 0.2 + 0.6 × E
```

### 失真倾向
```
distortion = 0.1 + 0.4 × N + 0.3 × (1 - A)
```

## 5. Edge Cases

- **无目击者神迹**：神迹发生在无人区域 → 无直接感知，可能通过痕迹间接传播
- **高宜人性村庄**：所有 A 值高 → 失真极低 → 信仰高度一致（实验对照组）
- **孤立村民**（E 极低）：长期不参与传播 → 信仰独立演化，形成异端
- **连续神迹**：前一神迹尚未消化时施放新神迹 → 两种神话相互影响/冲突

## 6. Dependencies

- `backend/agents/agent.py` — 五层 Agent 核心
- `backend/memory/memory_stream.py` — 记忆系统
- `backend/propagation/distortion.py` — 失真引擎
- `backend/agents/village.py` — 村落仿真器
- `backend/server.py` — FastAPI 后端

## 7. Tuning Knobs

| 参数 | 默认值 | 范围 | 影响 |
|------|--------|------|------|
| VILLAGE_SIZE | 8 | 3-20 | API 成本、社会网络复杂度 |
| TICK_INTERVAL | 5s | 1-60s | 游戏节奏 |
| PROXIMITY_RADIUS | 3格 | 1-6 | 村民相遇频率 |
| MEMORY_RECENCY_DECAY | 0.995 | 0.98-0.999 | 记忆遗忘速度 |
| MEMORY_IMPORTANCE_THRESHOLD | 150 | 50-300 | 反思频率 |
| MIRACLE_COOLDOWN | 15s | 5-60s | 玩家干预频率 |

## 8. Acceptance Criteria

**可玩性标准：**
- [ ] 玩家可在 5 分钟内施放首个神迹并看到村民对话反应
- [ ] 施放相同神迹两次，因世界状态不同产生不同解读
- [ ] 10 tick 内至少 1 名村民形成信念（L4 层触发）

**研究标准（P1/P2）：**
- [ ] 语义失真随传播跳数单调递增（可量化验证）
- [ ] N 维度高的 Agent 平均失真 > N 维度低的 Agent
- [ ] 每次仿真可导出完整 distortion_records JSON 供 Jupyter 分析

**技术标准：**
- [ ] Godot ↔ Python 延迟 < 500ms
- [ ] 10 tick 仿真 API 成本 < ¥0.5（DeepSeek V3）
- [ ] 支持离线模式（Ollama 替换 DeepSeek）
